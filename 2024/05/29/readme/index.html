读者功能

读者账号登录

在用户入口界面user网页中，输入用户账号和密码即可进行账号登陆操作，后台会返回两种结果：

  • 登录成功，登录成功后用户可以进行图书借阅。
  • 登录失败，页面会提示登录失败的原因：账号密码输入错误或用户被锁定。
读者退出登录

当读者登录后，在user界面可以点击退出按钮，即可退出登录的状态。

读者修改账号密码

在登录之后，用户如果想修改密码，可以点击左侧修改密码按钮进入user/change-password页面，即可进行修改密码的操作，有如下情况：

  • 两次新密码不一致：页面内通过js识别并提示出两次输入不一样，禁止修改密码。
  • 旧密码不正确：在判定之后返回修改页，提示密码错误。
  • 修改成功：然会user登录界面。
校外人员注册账号

对于校外读者，可以自己进行注册，点击左侧校外用户注册按键，进入user/signup界面。填写账号信息和密码注册之后回到初始登录界面，然后就可以用注册的账号登录了；如果注册信息有问题则返回注册页面。

图书信息查询

在左侧点击图书信息查询按键进入user/search界面，这个界面里可以根据关键词查找匹配有关的书，按照5个一页的方式展示在页面中。如果想要重新搜索，可以点击链接返回到搜索界面。

图书借阅

在图书查找页面user/search查找图书的过程中，可以通过点击借书按钮，跳转到user/borrow页面，把某书添加到借阅列表 ，然后可以点击左侧看看有没有其它书继续借阅，或者点击确认借书。

图书归还&延期归还

在图书归还页面user/return可以归还已经借阅的图书，或者续借图书（最长续借7天）。

通过邮箱与管理员联系

如果有任何疑问，可以在主界面重要信息栏点击邮箱向管理员发送邮件。

数据库设计

遇到的问题

已经解决的

  • 数据库的驱动问题。

解决方法 ：利用ODBC驱动，此外还可以利用python的flask_sqlalchemy包来解决，但是由于flask_sqlalchemy包的sql语句又需要另外的语法来解决，最终选择了ODBC驱动的方法。

  • blueprints中循环引用报错的问题。

解决方法 ：利用一个utils.py文件，将共同引用的部分放入这个文件中，再导入解决。

  • 如果所有函数都写在app.py中则代码非常冗余，维护起来很困难。

解决方法 ：利用blueprint，将代码划分成user和admin两部分，分别对应/user/和/admin/两个路由，大大增强了代码的可读性。

  • 蓝图的注册、静态路由、视图函数的用法。

解决方法 ：主要参考了python flask 蓝图(Blueprint)详解_flask python blueprint-CSDN博客、python框架之Flask(3)-Blueprint(蓝图)-CSDN博客这两篇blog。

  • 对于上锁的用户如何限制其登录。

解决方法 ：我们创造了一个user_login_form路由，当登陆中读取到用户被锁定后，跳转到这个路由，并接受error的参数，并将error参数传送给user_login.html文件，在网页上渲染被锁无法登录的信息。

还没有解决的

  • 在极少情况下会网页出现error状况。

未来的规划

  • 设计更多类型的管理员。

  • 用户注册时使用邮箱才可以注册；并且判断用户密码的安全等级，不能是很简单的密码。

  • 建立一个论坛，让老师同学们交流。

  • 建立推荐书目的窗口，根据同学们的借阅和检索数据推荐书目。
